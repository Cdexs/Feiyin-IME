## 1. 产品概述

Voice IME 是一款运行在 Windows 平台的轻量级语音输入法辅助工具。用户按下热键后，对着麦克风说话，系统自动完成「语音 → 文字 → 优化 → 注入」全流程，将最终文本写入当前聚焦的输入框。

### 1.1 产品定位

| 维度 | 描述 |
|------|------|
| 目标用户 | 日常在 Windows 上频繁打字的知识工作者、写作者、程序员 |
| 核心价值 | 降低打字负担，让语音输入可以无缝替代键盘输入 |
| 差异化 | 本地推理（无隐私风险）+ AI 优化（输出更自然）+ 自动学习（越用越准） |

### 1.2 设计原则

- **低感知**：常驻系统托盘，不干扰其他应用
- **低延迟**：目标端到端延迟（语音停止 → 文字注入）≤ 3s（本地模型）
- **低资源**：后台待机内存 ≤ 80MB，CPU 占用 < 1%
- **可配置**：热键、LLM、词库均可用户自定义

## 2. 功能需求

### 2.1 核心功能

#### F-01：语音录制
- **F-01-1**：用户按下配置的热键，开始麦克风录音
- **F-01-2**：系统检测到持续静音（默认 1.2s）后自动停止录音
- **F-01-3**：支持两种热键模式：
  - **切换模式（Toggle）**：按一下开始录音，再按一下停止
  - **按住说话模式（Push-to-Talk）**：按住录音，松开停止
- **F-01-4**：录音最长上限 60 秒（可配置），超时自动停止

#### F-02：本地语音转录
- **F-02-1**：使用本地 Whisper Base 模型对录音进行转录，输出文本
- **F-02-2**：首次运行时自动下载模型文件（~142MB），缓存至本地
- **F-02-3**：支持转录语言：zh / en / ja / ko / auto（自动检测）

#### F-03：LLM 文本优化
- **F-03-1**：转录完成后，将文本提交至用户配置的 OpenAI 兼容接口进行优化
- **F-03-2**：优化目标：修正错别字、调整语序、补全标点、Markdown 格式化
- **F-03-3**：若 LLM 不可用（未配置/网络错误），降级使用原始转录文本
- **F-03-4**：LLM 调用最多重试 3 次（网络/超时错误才重试），重试间隔 600ms；单次超时 20s；重试耗尽后立即降级

#### F-04：文本注入
- **F-04-1**：将最终文本注入当前系统焦点所在的输入框
- **F-04-2**：优先使用剪贴板 + Ctrl+V 方式注入（速度快、Unicode 安全）
- **F-04-3**：剪贴板注入失败时自动降级为 SendInput 逐字符模拟
- **F-04-4**：注入完成后恢复用户原有剪贴板内容（延迟可配置，默认 150ms）
- **F-04-5**：若录音开始后目标窗口失去焦点，注入完成后弹出预览卡片，用户可复制文本

#### F-05：用户词库
- **F-05-1**：用户可在设置界面手工添加词对（原文 → 替换词）
- **F-05-2**：转录文本在 LLM 优化前，先经过词库替换
- **F-05-3**：【待实现】注入文本后进入 300ms 观察窗口，检测用户手动修改
- **F-05-4**：【待实现】若检测到修改，自动将差异词对写入词库（自动学习）
- **F-05-5**：用户可在设置界面查看词条（按使用次数降序）、删除词条

#### F-06：配置管理
- **F-06-1**：设置界面支持配置：LLM API 地址、API Key、模型名称、System Prompt、启用开关
- **F-06-2**：设置界面支持配置：录音热键（从预设列表选择：F9/F10/Ctrl+Space/Alt+`/Ctrl+Alt+V）
- **F-06-3**：设置界面支持配置：静音阈值、静音停止时长、最长录音时间
- **F-06-4**：设置界面支持配置：注入方式（剪贴板 / SendInput）、剪贴板恢复延迟
- **F-06-5**：配置持久化到 `%APPDATA%\voice-ime\config.toml`
- **F-06-6**：当转录语言为中文时，支持选择输出字形（简体 / 繁体）
- **F-06-7**：录音覆盖层透明度可配置（30%–100%，默认 75%）

#### F-07：系统托盘
- **F-07-1**：程序以系统托盘图标常驻后台，静默启动（不显示主界面）
- **F-07-2**：托盘图标 tooltip 反映当前状态（待机 / 录音中 / 处理中）；【待实现】图标图像随状态变化
- **F-07-3**：右键托盘菜单提供：打开设置、退出
- **F-07-4**：双击托盘图标打开设置窗口

#### F-08：双语界面
- **F-08-1**：设置界面支持中文与英文两种显示语言
- **F-08-2**：语言切换立即生效，无需重启

#### F-09：录音覆盖层（Overlay）
- **F-09-1**：按热键时，在屏幕底部中央弹出无边框半透明录音条
- **F-09-2**：录音条内含：状态点动画、音频波形、取消按钮
- **F-09-3**：支持按 Escape 键或点击取消按钮终止当前录音/处理
- **F-09-4**：录音/处理完成后，覆盖层自动消失
- **F-09-5**：覆盖层与设置窗口相互独立，可同时存在

### 2.2 非功能需求

| 编号 | 类型 | 要求 |
|------|------|------|
| NF-01 | 性能 | 语音停止 → 文字注入（含本地推理）延迟 ≤ 3s |
| NF-02 | 资源 | 后台待机内存 ≤ 80MB，CPU < 1% |
| NF-03 | 可靠性 | 单步骤失败不应导致程序崩溃，应降级处理 |
| NF-04 | 隐私 | 所有语音转录在本地完成，不上传音频数据 |
| NF-05 | 平台 | 支持 Windows 10（1903+）及 Windows 11 |
| NF-06 | 安装 | 提供 Inno Setup 安装程序，支持开机自启选项 |

### 2.3 排除范围（当前版本）

- 多语言实时切换（转录时切换语言）
- 云端模型推理
- 移动端 / macOS / Linux
- 多用户/多配置文件
- 实时流式转录反馈
- 托盘图标动画（状态图像切换）

## 3. 用户交互流程

```
[用户按热键]
    │
    ▼
[录音覆盖层弹出 — 底部中央]
[托盘图标变为"录音中"]
    │
    ├─ [用户按 Escape / 点取消] → [取消，覆盖层消失]
    │
    ▼
[检测到静音 or 再次按热键 or 超时]
    │
    ▼
[覆盖层显示"转录中…"]
[本地 Whisper 转录]
    │
    ▼
[词库替换]
    │
    ▼
[覆盖层显示"优化中…"]
[LLM 优化，最多重试 3 次]  ──失败──► [使用原始文本，覆盖层短暂提示]
    │
    ▼
[检查焦点窗口]
    ├─ 焦点存在 → [覆盖层显示"注入中…"] → [注入到输入框] → [覆盖层消失]
    │
    └─ 焦点丢失 → [弹出结果预览卡片] → [用户点复制按钮] → [卡片消失]
    │
    ▼
[托盘图标回到"待机"]
```

## 6. 验收标准（当前构建）

- [ ] 按热键录音，录音覆盖层弹出（底部中央，半透明）
- [ ] 语音内容被正确转录为中文文本
- [ ] 转录文本经 LLM 优化后注入记事本输入框
- [ ] LLM 未配置时，原始文本也能正确注入
- [ ] 设置界面可保存 LLM 配置，重启后配置保持
- [ ] 手工添加词库条目，该词在下次转录中被替换
- [ ] 按 Escape 或点击取消按钮，录音中止，覆盖层消失
- [ ] 焦点丢失时弹出预览卡片，点复制后卡片消失
- [ ] 右键托盘 → 退出，程序完全退出
- [ ] 右键托盘 → 设置，设置窗口正常弹出并获得焦点
- [ ] 程序运行时内存 ≤ 80MB（不含 Whisper 模型加载时）
